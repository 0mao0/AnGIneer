# SOP 经验库页面重构设计

> 日期：2026-05-11
> 状态：已确认，待实施

## 1. 目标

本次重构聚焦 `apps/admin-console` 中的经验库管理页，目标不是重新设计一套新交互，而是直接复用知识库与测试集已经验证过的成熟模式。

重构后的经验库页面需满足以下要求：

- 左侧树采用与知识库、测试集几乎一致的管理态模式，补齐搜索、根级新增、文件夹/文件节点操作、拖拽移动与空状态入口。
- 右侧详情区保持 `属性` 与 `AI 对话` 两个页签，但页签视觉更轻、更紧凑，整体满高，不再出现右栏整体纵向滚动条。
- `AI 对话` 直接使用共享 `AIChat`，不在页面层重复实现上下文统计、清空按钮等平行 UI。
- `属性` 面板改为紧凑布局，减少纵向浪费，并将 `备注` 并入 `描述`。
- SOP 步骤 JSON 结构升级为 `execution` 分组结构，同时兼容旧数据读取。

## 2. 范围

本次改造包含：

- 页面编排与布局调整。
- 左侧经验树管理态能力补齐。
- 右侧属性面板重构。
- `SopStep` 类型与序列化结构调整。
- 新建空白 SOP 与导入 JSON 两类新增文件能力。

本次改造不包含：

- 流程图画布交互模型重做。
- `AIChat` 通用能力的大范围改版。
- SOP 后端协议无关的额外重构。

## 3. 总体方案

### 3.1 页面结构

经验库页面继续维持三栏结构：

- 左侧：经验树，负责 SOP 文件与文件夹管理。
- 中间：流程图画布，保持当前主工作区职责。
- 右侧：详情区，包含 `属性`、`AI 对话` 两个页签。

页面结构不再追求新增复杂抽象，而是在现有 `ExperienceManage.vue` 基础上，按知识库与测试集的成熟案例对齐。

### 3.2 复用原则

- 左树继续使用 `SOPTree.vue` 作为经验域语义组件，不直接在页面中裸用 `SmartTree`。
- 管理态交互直接对齐 `KnowledgeManage.vue` 与 `EvalManage.vue` 的使用模式，包括搜索、根级新增、节点操作、拖拽与空状态。
- 右侧 AI 面板只挂载 `AIChat.vue`，页面层不再重复封装头部功能。

## 4. 左侧树设计

### 4.1 交互模式

左侧树直接套用知识库/测试集已验证的管理态模式，经验库只替换节点语义为 SOP：

- 顶部搜索框：支持按 SOP 名称与文件夹名称过滤。
- 根级操作：支持 `新建文件夹`、`新建 SOP`、`导入 JSON`。
- 文件夹节点操作：支持 `重命名`、`新增子文件夹`、`新增 SOP`、`导入 JSON`、`删除`。
- 文件节点操作：支持 `重命名`、`查看`、`删除`。
- 拖拽操作：支持拖入文件夹、同级排序、拖到根目录，并继承 SmartTree 的非法拖拽约束。
- 空状态：提供最小但完整的入口，不再只保留单一“大按钮”。

### 4.2 数据流

`useSopTree.ts` 扩展为管理态 composable，负责：

- 拉取 SOP 列表与文件夹结构并构建树。
- 创建空白 SOP。
- 导入 JSON 为 SOP。
- 创建/重命名/删除文件夹。
- 删除/重命名/定位 SOP。
- 刷新后自动聚焦新建或导入节点。

页面层只负责编排与交互触发，不承载大量树管理分支逻辑。

## 5. 右侧详情区设计

### 5.1 页签样式

右侧保留 `属性`、`AI 对话` 两个页签，但要从默认重视觉 `a-tabs` 切换为更轻、更紧凑的页签头。目标是弱化“组件默认样式感”，让右栏更像一个工作侧边栏。

### 5.2 满高与滚动策略

右侧详情区、页签容器、属性内容区、AI 内容区全部建立 `height: 100%` 链路。

滚动策略如下：

- 页面容器不出现额外纵向滚动条。
- 右栏整体不滚。
- 仅属性内容区在内容过多时允许局部滚动。
- `AIChat` 自身消息区按组件既有方式内部滚动。

## 6. 属性面板设计

### 6.1 布局原则

属性面板不再使用当前“一行一个 label + form-item”的纵向堆叠形式，而改为紧凑编辑布局：

- `步骤 ID` 作为轻量只读信息展示，不再占用标准表单行。
- `名称` 不显示显式 label，直接使用输入框表达语义。
- `描述` 作为主要文本编辑区，并吸收原 `备注` 内容。
- `工具 + 输入参数 + 输出` 作为一个完整的“执行定义”分组连续呈现。
- 面板底部补齐明确的 `取消`、`保存` 按钮。

### 6.2 编辑模型

属性面板采用“局部草稿 + 显式保存”模式：

- 进入步骤时从当前步骤数据生成本地草稿。
- 编辑过程中仅更新草稿，不立即污染流程图源数据。
- 点击 `保存` 时统一提交到页面，再同步到 `useSopFlow`。
- 点击 `取消` 时恢复为当前步骤最近一次已加载值。

这能解决当前只有即时更新、没有取消/保存按钮的问题。

## 7. SOP JSON 结构升级

### 7.1 新结构

步骤结构从平铺字段升级为执行块分组结构：

```json
{
  "id": "step_2",
  "name": "计算各重现期波浪超高 Δw",
  "description": "对每个重现期，计算波浪超高 Δw ...\n\n参考第5.4.8.1款和第5.4.8.2款。",
  "execution": {
    "tool": "calculator",
    "inputs": {
      "expression": "hs + H_wave / 2",
      "variables": "${variables}"
    },
    "outputs": {
      "delta_w_basic": "result",
      "delta_w_extreme": "result"
    }
  }
}
```

### 7.2 兼容策略

旧结构：

- `tool`
- `inputs`
- `outputs`
- `notes`

读取时前端先归一化：

- 若存在 `execution`，直接使用新结构。
- 若不存在 `execution`，则从旧字段组装出运行时 `execution`。
- 若存在 `notes`，则按“描述在前、备注追加在后”的规则合并进 `description`。

保存时统一写回新结构，不再写出 `notes` 与平铺 `tool/inputs/outputs`。

## 8. 涉及文件

- `apps/admin-console/src/views/ExperienceManage.vue`
- `packages/sop-ui/src/components/SOPTree.vue`
- `packages/sop-ui/src/composables/useSopTree.ts`
- `packages/sop-ui/src/components/SOPPropertyPanel.vue`
- `packages/sop-ui/src/types/sop.ts`
- `packages/sop-ui/src/composables/useSopApi.ts`（按需补齐能力）

若确实需要补充一个极小型 SOP 新建入口，优先扩展现有交互，不新增平行管理组件。

## 9. 错误处理

- 左树新增、导入、删除、重命名、拖拽失败时，使用现有消息提示模式反馈错误。
- 新建 SOP 与导入 JSON 成功后，必须刷新树并尽量自动聚焦目标节点。
- 若旧 JSON 缺少 `description`、`execution` 或字段不完整，前端归一化时提供安全默认值，避免属性面板空引用报错。

## 10. 验证

实施后至少验证以下场景：

- 左树搜索、新建文件夹、新建 SOP、导入 JSON 可用。
- 文件夹与文件节点操作完整，行为与知识库/测试集管理态一致。
- 拖拽移动、拖到根目录、非法拖拽提示正常。
- 右侧详情区整体满高，无额外整体纵向滚动条。
- `AIChat` 区域直接复用共享组件，消息区正常滚动与输入。
- 属性面板支持取消与保存，且纵向空间占用明显下降。
- 旧结构 SOP 可正常读取，新结构保存结果正确。
- 前端诊断与必要 lint 检查通过。

## 11. 风险与约束

- 若现有 SOP 后端 API 尚未覆盖创建空白 SOP、导入 JSON、移动排序等能力，需要补最小必要协议，但不引入与知识库平行的新体系。
- 属性面板去掉显式 label 后，必须通过分组、placeholder 与只读信息层次保证可读性。
- `AIChat` 若仍需补共享能力，应优先沉淀到 `ui-kit`，而不是再次在经验库页做特例 UI。
