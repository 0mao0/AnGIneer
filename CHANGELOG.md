# 更新日志 (CHANGELOG)

所有对 PicoAgent 项目的显著更改都将记录在此文件中。

## [0.0.05] - 2026-01-26

### 🚀 新增 (Add)
- **tool测试**:初步完成tool的测试页面搭建。
- **表格tool构建**:改为纯硬编码的tool插值类，避免llm的解析时间过长。

### 🔧 优化 (Changed)
- **SOP缓存策略**: 更新了SOP缓存策略，将SOP缓存策略从SOP_json更改为SOP_cache，以适应新的缓存策略。


## [0.0.04] - 2026-01-26

### 🚀 新增 (Add)
- **SOP_json策略**:新增sop预分析，降低slm的实时解析sop的瓶颈（可能长达45s）。使用预分析后，可提升至6s。

### 🔧 优化 (Changed)
- **SOP 缓存策略**: 禁用了 LLM 解析结果自动回写到 JSON 文件的功能，以保护官方 JSON 文件的唯一性和纯洁性，便于测试与版本控制。
- **实时控制台**: 优化了前端测试控制台的输出逻辑，增加了 5 秒智能心跳机制。当无数据输出时显示心跳标记，有数据时自动抑制心跳，确保信息展示的连贯性与存活感。

## [0.0.03] - 2026-01-26

### 🚀 新增 (Added)
- **实时流式反馈**: 实现了 Test03 的分步骤实时渲染，每个步骤卡片现在独立计时并动态更新耗时。
- **异步分析优化**: 引入了 LLM 解析阶段的心跳推送机制，彻底解决了长耗时任务（如 SOP 解析）导致的界面阻塞问题。
- **分步计时系统**: 为 SOP 匹配、SOP 解析及每个执行步骤增加了高精度独立计时显示。

### 🔧 优化 (Changed)
- **UI/UX 改进**: 
  - 重新设计了 SOP 匹配卡片，将其作为独立节点展示，并支持子步骤缩进排版。
  - 优化了卡片信息层级，隐藏了冗余的分析描述，保留了核心的工具、输入输出与执行结果。
- **稳定性增强**: 优化了后端流式数据的封装与推送逻辑，确保浏览器能更平滑地处理分段 JSON 数据。

## [0.0.02] - 2026-01-25

### 🚀 新增 (Added)
- **前端模型选择**: 在测试面板中增加了模型选择下拉框，默认设置为 Qwen-VL。
- **推理时间显示**: 意图路由结果中新增推理耗时（Duration）显示。
- **流程图动态效果**: 运行中的流程图节点会自动居中并保持动态向下的视觉感。

### 🔧 优化 (Changed)
- **流程图稳定性**: 修复了 Mermaid 渲染中的括号解析错误，确保节点标签在包含特殊字符时能正确显示。
- **数据同步**: 修复了前端无法正确显示后端路由原因（Reason）的问题，完善了 `api_bridge.py` 与前端的数据对接。
- **GitHub 兼容性**: 修复了 `README.md` 中的 Mermaid 语法，使其在 GitHub 上能正常渲染。
- **忽略项更新**: 在 `.gitignore` 中增加了 `.trae/` 文件夹。

## [0.0.01] - 2026-01-18

### 🚀 新增 (Added)
- **混合架构基础**: 实现了规则执行与 LLM 决策相结合的混合 Agent 调度引擎。
- **SOP 智能加载器**: 支持 Markdown 格式 SOP 的自动解析与 LLM 预分析，提取输入参数与注意事项。
- **NVIDIA API 集成**: 默认支持 NVIDIA 提供的多种大模型（如 Nemotron-3-nano, DeepSeek-v3.2, Kimi 等）。
- **工具集**: 
  - 通用工具：计算器、智能查表 (Table Lookup)、知识检索 (RAG)。
  - GIS工具初试试：实现断面工程量体积计算。
- **双语支持**: 工具描述支持中英双语，代码注释与日志全面本地化。

### 🔧 优化 (Changed)
- **重构路由层**: 将 `router.py` 重命名为 `classifier.py`，优化意图识别逻辑与加载顺序。
- **项目文档**: 全面重写 README.md，包含架构图、路线图及快速开始指南。
